# Design System

The RankedTag visual language. Everything here is pulled from rankedtag.com so the deliverable feels like a natural extension of the brand.

## Palette

| Role | Hex | Use |
|------|-----|-----|
| Ink | `161618` | Primary text, hero/proof bands |
| Ink 2 | `2A2B33` | Secondary dark |
| Signature red | `FF3B14` | Accent, eyebrows, bullets, left rules, the brand pop |
| Red dark | `C8260A` | Red text on light (contrast-safe) |
| Periwinkle | `6B77E0` | Secondary accent, links, "why" callouts |
| Periwinkle light | `A6B0F0` | Accent on dark bands |
| Green | `2D8A5C` | "Good" / positive / "what to do" |
| Green dark | `1F4D3F` | Green text on light |
| Amber | `C97A06` | "Fair" status |
| Muted | `6E6E76` | Captions, secondary text |
| Paper | `F4EFE7` | Page-adjacent panels, the warm cream base |
| Paper 2 | `EDE6D9` | Empty track in bars |
| Line | `E0D7C7` | Hairlines, table borders |
| White | `FFFFFF` | Card fills |

Tint fills for callout boxes: red tint `FBEAE4`, green tint `E6F1EC`, periwinkle tint `EEF0FB`, amber tint `FBF0DD`.

Status → color mapping for scores: **≥75 green**, **50–74 amber**, **<50 red**.

## Type

Arial throughout (universally supported in docx). Default body 21 half-points (~10.5pt), line spacing ~276. Headings bold ink. Eyebrows are small-caps-feel: bold, ~17–20 half-pt, letter-spaced, usually red or a section accent color.

## Layout (docx)

- US Letter, 1-inch margins. Content width **9360 DXA**.
- Footer on every page: hairline rule, "RankedTag · SEO & AI-search audit for <client>" left, page number right (tab stop at 9360).
- **Hero/proof bands**: full-width single-cell table, `fill: INK`, generous margins (~340–420 DXA), no borders. White headline, periwinkle-light eyebrow, light-gray subtext.
- **Callout boxes**: single-cell table with a thick left border (`size: 28`) in the accent color, a tint fill, an uppercase letter-spaced label in the accent, then body. Used for "why this matters", "what to do", founder quote, "what this means for you".
- **Tables**: ink header row, white/paper zebra striping, `Line` borders, DXA widths that sum to the table width, cell margins ~80/130. Never `WidthType.PERCENTAGE`.
- **Eyebrow + numbered H1** pattern opens each major section: a red eyebrow line, then `N  Section title`.
- Embed visuals centered at ~624px wide (full content width) with an italic muted caption beneath. Screenshots: GSC chart ~624px, the AI-Overview SERP ~470px (it's taller).

## The four signature visuals

Generated by `scripts/generate_visuals.py`. All use the palette above, rounded corners, the cream base.

1. **Scorecard** (`viz_scorecard.png`) — left ink panel with the big overall score + FAIR/GOOD/POOR pill; right panel lists the 7 categories as horizontal bars colored by status band with the numeric score. This is the dashboard the client reads first.
2. **Rich result** (`viz_richresult.png`) — side-by-side SERP mockup: "TODAY — plain blue link" (red) vs "WITH SCHEMA — rich result" (green) showing stars, FAQ accordion, breadcrumb. Makes the #1 schema finding instantly legible.
3. **Meta before/after** (`viz_meta.png`) — the truncated meta description (red, cut off mid-sentence with an ellipsis) above the rewritten complete one (green, under 155 chars). Shows the fix, not just the problem.
4. **Roadmap** (`viz_roadmap.png`) — three columns: FIX NOW (red) / THIS WEEK (amber) / THIS MONTH (green), each listing its items. The 30-day sequence.

Glyphs like stars and carets are drawn as SVG shapes, not font characters (the sandbox font may not have them). The script already handles this.

After rendering, always `view` each PNG and confirm no text clips the panel edges. If a long category label or client name overflows, the script accepts flags to adjust, or shorten the label.
